# Icon Candidate Contract

This directory contains three distinct icon/logo candidate systems for the Year Planner app.
Each candidate provides canonical SVG masters from which preview PNGs and production assets are derived.

## Candidate Folder Contract

Each candidate lives in a folder named `C{N}-{slug}/` and must contain:

| File | Purpose | Dimensions |
|------|---------|-----------|
| `icon.svg` | Square app-icon master | `viewBox="0 0 512 512"` |
| `logo.svg` | Horizontal lockup master | `viewBox="0 0 480 120"` |
| `previews/` | Rasterised PNGs (generated by `scripts/export-icon-candidates.sh`) | see sizes below |

### Required Preview PNG Sizes

Each candidate must export the following PNGs inside its `previews/` folder:

| File | Use |
|------|-----|
| `icon-16.png` | Favicon 16×16 |
| `icon-32.png` | Favicon 32×32 |
| `icon-180.png` | Apple touch icon 180×180 |
| `icon-192.png` | PWA manifest icon 192×192 |
| `icon-512.png` | PWA splash / app store 512×512 |

## Naming Convention

| Segment | Rule |
|---------|------|
| Candidate ID | `C1`, `C2`, `C3` — stable, never renumbered |
| Slug | Kebab-case design-system name (e.g. `ink-paper`) |
| Folder | `C{N}-{slug}` (e.g. `C1-ink-paper`) |
| SVG masters | `icon.svg`, `logo.svg` — exact names, always lowercase |
| Preview PNGs | `icon-{size}.png` where `{size}` is the pixel dimension of one side |

## SVG Master Requirements

All master SVGs must satisfy:

- Valid `viewBox` attribute (no `width`/`height` attributes at root level — let the browser scale)
- Adequate padding so shape edges are not clipped when rasterised at small sizes
- No embedded raster images, no external href references, no `<use>` referencing out-of-file symbols
- `<title>` element immediately inside `<svg>` for accessibility
- Clean path geometry — avoid transforms that accumulate precision errors

## Candidates

| ID | Folder | Design Direction |
|----|--------|-----------------|
| C1 | `C1-ink-paper/` | Editorial warmth — Playfair-serif, warm cream paper, terracotta/amber accents |
| C2 | `C2-nordic-clarity/` | Scandinavian minimalism — geometric precision, dark header band, electric blue + coral |
| C3 | `C3-verdant-studio/` | Organic craft — forest green, organic arcs, warm stone, rust accent |

## Machine-Checkable Contract

The asset smoke spec (`.tests/smoke/icon-candidates-assets.spec.js`) enforces:

1. Exactly C1, C2, C3 candidate folders exist under `mockups/icon-candidates/`.
2. Each candidate folder contains `icon.svg` and `logo.svg`.
3. Each SVG file is non-empty and contains a `viewBox` attribute.
4. The `icon.svg` viewBox is `0 0 512 512`.
5. The `logo.svg` viewBox is `0 0 480 120`.
