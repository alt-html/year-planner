## Concrete Change Details

### 1. `js/vue/methods/entries.js`

**`updateEntryState`**: S01 did NOT add notes/emoji population here (verified against source). S02 must add:
```js
this.entryNotes = this.getEntryNotes(mindex, day);
this.entryEmoji = this.getEntryEmoji(mindex, day);
```

**`updateEntry`**: Current signature is `(mindex, day, entry, entryType, entryColour, syncToRemote)`. Extend to:
```js
updateEntry(mindex, day, entry, entryType, entryColour, notes = '', emoji = '', syncToRemote = false)
```
Update the `storageLocal.updateLocalEntry(...)` call to pass `notes, emoji` after `entryColour`.

### 2. `.compose/fragments/modals/entry.html`

Replace the single `<textarea v-model="entry" id="yp-entry-textarea" ...>` with three fields:

1. **Tagline** â€” `<input type="text" v-model="entry" id="yp-entry-textarea" maxlength="32" ...>` 
   - **Must keep `id="yp-entry-textarea"`** â€” the E2E test uses `page.fill('#yp-entry-textarea', ...)`.
   - Change from `<textarea>` to `<input type="text">` (tagline is â‰¤32 chars, single-line).

2. **Notes** â€” `<textarea v-model="entryNotes" id="yp-notes-textarea" rows="3" ...>` (long-form)

3. **Emoji** â€” `<input type="text" v-model="entryEmoji" id="yp-emoji-input" maxlength="4" ...>` (single emoji character)

All `updateEntry(...)` call sites in entry.html â€” currently 10 calls across colour dots + save button â€” need `entryNotes, entryEmoji` inserted before the final `true` (syncToRemote) arg:

- Colour dots: `updateEntry(month,day,entry,entryType,N,entryNotes,entryEmoji,true)`
- Save button: `updateEntry(month,day,entry,entryType,entryColour,entryNotes,entryEmoji,true)`

### 3. `.compose/fragments/grid.html`

Current cell: 3 `.yp-cell-text` spans â€” day number, `substr(0,20)` of tagline, `substr(20)` of tagline overflow.

**E2E constraint**: `day1Cell.locator('.yp-cell-text').nth(1)` must contain the tagline text. The test uses `toContainText` (substring match) so emoji prepended in the same span is fine.

Change to 2 spans:
```html
<span class="yp-cell-text">{{DateTime...toLocaleString({day:'numeric'})}}&nbsp;</span>
<span class="yp-cell-text"
      data-toggle="tooltip" data-placement="bottom"
      :title="(getEntryEmoji(mindex,n) ? getEntryEmoji(mindex,n)+' ' : '')+getEntry(mindex,n).trimStart()">
    {{(getEntryEmoji(mindex,n) ? getEntryEmoji(mindex,n)+' ' : '')+getEntry(mindex,n).trimStart()}}
</span>
```
CSS already handles `overflow: hidden; text-overflow: ellipsis` on `.yp-cell-text` â€” the `substr(20)` third span is a legacy pre-CSS hack and can go.

### 4. `.compose/fragments/scripts.html`

The marker mode call (line ~686 of index.html, inside `applyMarkerToCell`):
```js
// Before:
vueInstance.updateEntry(mindex, day, entry, entryType, markerColour, true);
// After (read current notes/emoji to avoid overwriting):
var notes = vueInstance.getEntryNotes(mindex, day);
var emoji = vueInstance.getEntryEmoji(mindex, day);
vueInstance.updateEntry(mindex, day, entry, entryType, markerColour, notes, emoji, true);
```
This preserves existing notes/emoji when the marker only changes colour.

### 5. `css/main.css` â€” Modal field styles

Need to add styles for the tagline `<input>` (currently styled for `<textarea>`) and the new notes/emoji fields. The existing `.yp-entry-text` style applies to a textarea â€” convert or duplicate for an `<input type="text">`.

Approach: keep `.yp-entry-text` class on tagline input (same visual treatment, remove `min-height`/`resize`), add `.yp-entry-notes` for the notes textarea, add `.yp-entry-emoji` for the emoji input (narrow, centred, large text).

Also add `.yp-entry-field-label` for section labels (optional but cleaner UX).
