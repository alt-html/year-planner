# S06: Integrated Verification and Sign-off — Research

**Date:** 2026-04-16

## Summary

This is a verification and sign-off slice to meet requirement R006. The work from S04 (Web/PWA Wiring) and S05 (Desktop Packaging) has already produced comprehensive automated smoke tests that cover the *structural* integrity of the new icon system. Those 86 tests confirm that all icon files exist, are referenced correctly in `index.html` and `manifest.json`, and that the desktop asset binaries (`.ico`, `.icns`) are well-formed.

The remaining work is to satisfy the "explicit visual spot checks on key sizes/surfaces" part of R006. This research recommends creating a simple, automated process to generate a visual "sign-off sheet" as an HTML page, which can be screenshotted to provide a permanent, reviewable artifact. This approach makes the manual check repeatable and scriptable.

## Recommendation

Create a new Playwright test spec dedicated to visual verification. This spec will not assert correctness but will instead generate a webpage that curates all the key visual assets for easy review. This provides a single surface for a human to perform the visual sign-off.

The resulting screenshot of this webpage will serve as the primary deliverable and proof for this slice.

## Implementation Landscape

### Key Files

-   **`.tests/verification/S06-visual-sign-off.spec.js`** (New file): A Playwright test that will:
    1.  Read `site/icons/matrix.json` (for web/PWA icons) and `site/icons/desktop-matrix.json` (for desktop icons).
    2.  Generate an HTML string that displays each icon with a descriptive label (e.g., "PWA Maskable 512x512", "Windows ICO (favicon)"). Browsers can render `.ico` files directly in `<img>` tags. `.icns` files cannot be rendered directly, so for the macOS icon, the test will display the constituent PNGs that were used to create the `.icns` file, as generated by the export script logic.
    3.  Write this HTML to a temporary file, for example, `.tests/verification/S06-sign-off-sheet.html`.
    4.  Use Playwright to navigate to this local HTML file.
    5.  Take a full-page screenshot and save it as an artifact, e.g., `.tests/verification/S06-visual-sign-off-sheet.png`.
-   **`site/icons/matrix.json`** (Input): Source of truth for web, PWA, and favicon assets.
-   **`site/icons/desktop-matrix.json`** (Input): Source of truth for desktop packaging assets.

### Build Order

The work is self-contained within a single new test file.
1.  Create the new spec file `S06-visual-sign-off.spec.js`.
2.  Implement the logic to read the two matrix files.
3.  Implement the HTML generation logic.
4.  Add the Playwright steps to write the file, navigate, and take a screenshot.

### Verification Approach

The slice is complete when:
1.  The new verification spec runs successfully without errors: `npx playwright test .tests/verification/S06-visual-sign-off.spec.js`.
2.  A screenshot artifact is successfully generated.
3.  The full existing test suite continues to pass, ensuring no regressions were introduced: `cd .tests && npx playwright test`. The expected result is 216 passing tests.
4.  The generated screenshot is presented as the evidence for completing the "visual spot check" requirement.
