# Requirements: Year Planner

**Defined:** 2026-03-12
**Core Value:** Offline-first local planner that works without an account, and syncs bidirectionally when signed in — without data loss across devices.

## Validated Requirements

### Sync Protocol

- [x] **AUTH-06**: Client-side sync layer uses jsmdma protocol — `POST /year-planner/sync` with HLC-clocked dot-path fieldRevs, `clientClock`, `changes` array, and `serverChanges` response. StorageRemote.js retired.
- [x] **SYNC-04**: Editing any day entry (tp, tl, col, notes, emoji) writes a dot-path HLC entry to `rev:{uuid}` in localStorage.
- [x] **SYNC-05**: POST /year-planner/sync endpoint wired end-to-end: SyncClient builds payload, Api.sync() calls it, all 9 Vue/Storage call sites updated.
- [x] **SYNC-06**: `SyncClient.js` implemented — wraps jsmdma sync protocol, manages baseClock/fieldRevs/baseSnapshot per planner, exposes sync()/markEdited()/prune(), CDI-registered as `syncClient`.

### Code Modernisation

- [x] **MOD-03**: StorageRemote.js deleted. All synchroniseToLocal/synchroniseToRemote references removed from 5 Vue method files and Storage.js.
- [x] **MOD-05**: SquareUp payment integration removed (SquareUp.js, pay.html modal, all references).
- [x] **MOD-06**: Donate feature flag and `window.ftoggle` global removed from model-features.js.
- [x] **MOD-07**: All 8 lodash (`_.filter`, `_.find`, etc.) calls replaced with native Array methods.

## Active Requirements

### Cleanup

- [x] **MOD-09**: Orphan modal fragment audit — confirmed 5 active fragments, 7 orphans deleted. Verified Phase 12.
- [x] **SYNC-08**: SyncClient.prune() wired to planner deletion lifecycle — rev/base/sync keys removed on delete. Verified Phase 12.

## Deferred

### Style

- **MOD-08**: Update Vue template bindings from `v-bind:`/`v-on:` to `:`/`@` shorthand across 6 fragment files.
  - *Deferred*: Vue 3 supports both forms identically. 41× v-bind: and 27× v-on: in index.html are harmless. Cosmetic-only change risks typos with zero functional benefit.

## Out of Scope

| Feature | Reason |
|---------|--------|
| Real-time collaboration | Not in product scope; offline-first sync is the model |
| Mobile native app | Web PWA is the target; mobile web covers the use case |
| SquareUp donations | Removed in Phase 2; dead code eliminated |
| Separate OpenAPI spec | jsmdma IS the sync API — data-api-core is the living contract (D004) |
| Vue Composition API migration | Options API works; migration would touch every reactive reference for zero user benefit |
