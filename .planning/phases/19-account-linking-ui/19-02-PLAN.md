**1. site/js/auth/AuthService.js** — Add `linkProvider(provider)` and `completeLinkCallback(provider, code, state)` methods after `unlinkProvider()`:

```javascript
    // Initiate OAuth link flow — redirects browser to provider (LNK-01)
    // Mirrors OAuthClient.signIn() but stores link intent flag
    async linkProvider(provider) {
        const res = await fetch(`${this._getApiUrl()}auth/${provider}`);
        if (!res.ok) throw new Error(`Server returned ${res.status}`);
        const { authorizationURL, state } = await res.json();
        localStorage.setItem('oauth_link_intent', provider);
        localStorage.setItem('oauth_link_state', state);
        window.location.href = authorizationURL;
        return new Promise(() => {});
    }

    // Complete link callback — calls POST /auth/link/:provider with OAuth code (LNK-01)
    // Returns updated provider names array, e.g. ['github', 'google']
    async completeLinkCallback(provider, code, state) {
        const token = this.getToken();
        if (!token) throw new Error('Not signed in');
        const storedState = localStorage.getItem('oauth_link_state') || '';
        const url = `${this._getApiUrl()}auth/link/${provider}?` +
            `code=${encodeURIComponent(code)}&` +
            `state=${encodeURIComponent(state)}&` +
            `stored_state=${encodeURIComponent(storedState)}`;
        const res = await fetch(url, {
            method: 'POST',
            headers: { 'Authorization': `Bearer ${token}` },
        });
        // Clean up link intent flags regardless of outcome
        localStorage.removeItem('oauth_link_intent');
        localStorage.removeItem('oauth_link_state');
        if (res.status === 409) throw new Error('error.providerConflict');
        if (!res.ok) throw new Error(`HTTP ${res.status}`);
        const { user } = await res.json();
        return user.providers.map(p => p.provider);
    }
```

**2. site/js/Application.js** — In `init()`, add link callback detection BEFORE the existing `urlToken` block (before line 36). The link callback arrives with `?code=` and `?state=` (NOT `?token=`), so it must be checked first:

```javascript
        // Handle OAuth link callback: detect oauth_link_intent + ?code= + ?state= (LNK-01)
        const linkIntent = localStorage.getItem('oauth_link_intent');
        const urlCode = urlParam('code');
        const urlState = urlParam('state');
        if (linkIntent && urlCode && urlState) {
            // Async link completion — sets model.linkedProviders after POST succeeds
            this._pendingLink = this.authProvider.completeLinkCallback(linkIntent, urlCode, urlState)
                .then(providers => {
                    this.model.linkedProviders = providers;
                    // LNK-04: migrate userKey on all local planners to primary UUID.
                    // plannerStore is accessed via this.model.plannerStore — CDI property-injects
                    // the PlannerStore singleton onto the model plain object (see model.js line 15).
                    // Application does NOT have plannerStore as a constructor param.
                    const plannerStore = this.model.plannerStore;
                    if (plannerStore) {
                        const planners = plannerStore.listPlanners();
                        for (const { uuid } of planners) {
                            plannerStore.takeOwnership(uuid);
                        }
                    }
                    this.logger?.debug?.(`[Application.init] link complete — providers=${providers.join(',')}`);
                })
                .catch(err => {
                    this.logger?.debug?.(`[Application.init] link failed: ${err.message}`);
                    this.model.modalError = err.message || 'error.general';
                });
            // Clean up URL query params
            const cleanUrl = new URL(window.location.href);
            cleanUrl.searchParams.delete('code');
            cleanUrl.searchParams.delete('state');
            window.history.replaceState({}, '', cleanUrl.toString());
        }
```

This block must appear BEFORE the `urlToken` block because:
- Sign-in callbacks arrive with `?token=` (server already exchanged the code)
- Link callbacks arrive with `?code=` + `?state=` (SPA must POST to /auth/link/:provider)
- When `oauth_link_intent` is set, we know this is a link callback, not sign-in

The `_pendingLink` promise is fire-and-forget from init()'s perspective. The model updates reactively when it resolves.

    async doLinkProvider() {
        const linked = this.linkedProviders || [];
        const available = this.availableProviders || [];
        const unlinked = available.filter(p => !linked.includes(p));
        if (unlinked.length === 0) return;
        // If only one unlinked provider, link it directly; otherwise link the first available

            this.modalError = err.message || 'error.general';

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> OAuth provider | Browser redirect to external OAuth consent page |
| client -> backend API | POST /auth/link/:provider with code, state, stored_state |
| OAuth callback -> SPA | Provider redirects back with ?code= and ?state= |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-01 | Spoofing | POST /auth/link/:provider | mitigate | OAuth state parameter validated server-side (`stored_state` must match `_pendingAuth` one-time entry); Bearer token required |
| T-19-02 | Tampering | oauth_link_intent localStorage | accept | Attacker would need physical access to localStorage; worst case is link attempt fails with 401/409 on server |
| T-19-03 | Repudiation | Link/unlink operations | accept | No audit trail beyond server-side user record; low-value target for non-enterprise app |
| T-19-04 | Information Disclosure | OAuth code in URL | mitigate | URL query params cleaned immediately via `history.replaceState`; code is single-use on server |
| T-19-05 | Denial of Service | Unlink last provider | mitigate | Server returns 409; client hides Unlink button when `linkedProviders.length <= 1` -- dual guard |
| T-19-06 | Elevation of Privilege | Link to another user's provider | mitigate | Server returns 409 if provider already linked to different user; no silent auto-linking |
</threat_model>
