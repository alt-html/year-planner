```javascript
async _signInGitHub() {
    const apiUrl = this._getApiUrl();
    let beginResult;
    try {
        const res = await fetch(`${apiUrl}auth/github`);
        if (!res.ok) throw new Error(`Server returned ${res.status}`);
        beginResult = await res.json();
    } catch (err) {
        throw new Error(`GitHub sign-in failed: could not reach auth server (${err.message})`);
    }
    const { authorizationURL } = beginResult;
    if (!authorizationURL) throw new Error('GitHub sign-in failed: no authorizationURL from server');
    localStorage.setItem('oauth_intended_provider', 'github');
    window.location.href = authorizationURL;
    return new Promise(() => {});
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| localStorage read -> auth_provider write | oauth_intended_provider value from localStorage is user-controllable; must be validated before storing as auth_provider |
| OAuth redirect return -> token handling | Token arrives as URL parameter from server redirect; already handled by existing ClientAuthSession.store |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-01 | Tampering | Application.js oauth_intended_provider read | mitigate | Validate against VALID_PROVIDERS array before writing to auth_provider: `const VALID_PROVIDERS = ['google', 'github', 'apple', 'microsoft']; const intendedProvider = VALID_PROVIDERS.includes(raw) ? raw : 'google';` |
| T-17-02 | Spoofing | Stale oauth_intended_provider from abandoned flow | mitigate | Clean up oauth_intended_provider in AuthProvider.signOut(), and overwrite on every new signIn call (each _signIn* method writes fresh value before redirect) |
| T-17-03 | Information Disclosure | Token visible in URL before replaceState | accept | Already mitigated in existing Application.js — replaceState strips ?token= immediately after storing |
| T-17-04 | Tampering | PKCE state in localStorage (vs sessionStorage) | accept | PKCE state (oauth_state, oauth_code_verifier) is verified server-side during token exchange; localStorage tampering cannot bypass server validation. Moving to localStorage is required for redirect flow survival. |
</threat_model>
